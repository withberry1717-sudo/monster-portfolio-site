Game Overview

作品概要

作品概要

『Monster Destroyer』は、プレイヤーが巨大な結晶ドラゴンに挑む 短編3Dボスバトルアクションです。 プレイヤーはブリンク、コンボ、ロックオン、尻尾切断などを活用しながら、 短時間でも濃いボス戦体験が得られることを目指しました。

本作では、「結晶をまとったドラゴン」というコンセプトを核に、 一定ダメージで尻尾を切断できるギミックを実装しました。 見た目の変化だけでなく、ボス戦の達成感を高める目玉要素として設計しています。

また、クリア時にはスコアとランクを表示し、 プレイヤーが自分のプレイを振り返り、より高いスコアを目指して再挑戦したくなるようにしました。 短編作品でもリプレイ率を上げることを意識しています。

Production Setup

制作条件と進め方

最初に工期を決めて制作

工期は最初から約2週間と定め、 そのうち1週間でプロトタイプ実装まで持っていくことを明確な目標にして制作しました。

ほぼ未経験の2人で挑戦

自分もデザイナーも、ゲーム制作・モデリングともにほぼ未経験の状態からスタートしました。 「限られた期間の中でどこまで完成度を高められるか」をテーマに進めました。

現実的な方法を調査して実行

自分が主導し、期限内に間に合わせるための現実的な制作手順を2人で調べ、 必要な作業を切り分けながら進行しました。

担当分担を明確化

デザインは自分とデザイナーで話し合って決定し、 技の構想は自分が担当、アニメーションの実装はデザイナーに担当してもらいました。

リグ遅延時は並列で進行

リグ入れが想定より難航した際は、私は先にプレイヤーアクションのブラッシュアップを進め、 制作の停滞を避けるようにしました。

7日で当たり判定・HPまで実装

実際に7日でプレイ可能な状態にし、 プレイヤーとボスの当たり判定、HP、戦闘の基本ループまで実装しました。

Difficulty Design

難易度ごとのクエスト演出

Easy / Normal / Hard の3段階を実装しました。
同じモンスターを使っていても単なる使い回しに見えないよう、 行動間隔・アニメーション速度・モンスターの大きさ・天候・雨パーティクルなどを調整し、 難易度ごとに異なる印象を作ることにも注力しました。

Easy

初心者向けの導入

初心者が遊びやすい入り口として設計。 ボスのサイズやテンポも含めて、まず行動を把握しやすい体験を目指しました。

Normal

標準体験の基準

本作の基準難易度です。 プレイ体験・スコア設計・UI・敵行動などの全体バランスは、 この難易度を中心に調整しました。

Hard

経験者向けの歯ごたえ

行動間隔やアニメーション速度を上げ、 雨パーティクルや天候変化も加えて、強敵感と緊張感を演出しました。

難易度調整で意識したこと

  • 同じボスでもサイズ・行動間隔・アニメーション速度を変えて体感差を出す
  • 天候や雨パーティクルで視覚的にも難易度の印象を変える
  • 短工期でも「ただ数値を変えただけ」に見えないよう演出面を追加する
  • 初心者でも楽しめる一方、経験者には歯ごたえが残る設計を目指す

Production Process

制作工程記録

結晶をまとったドラゴンと、それに対応するステージをオリジナルで構想・実装しました。
デザインは自分とデザイナーで話し合って方向性を決め、 ラフ → AI支援によるメッシュ生成 → リグ入れ → アニメーション → Unity実装 という流れで進めました。

① コンセプト決定

「結晶をまとったドラゴン」というコンセプトを2人で話し合い、 そこからデザイナーがラフを制作。ステージのラフも同時に作成しました。

② AIを使った三面図・プレビュー・メッシュ制作

ラフをもとにAIで三面図や3Dモデル風のプレビュー画像を作成し、 イメージ共有と方向性の確認に活用しました。メッシュ制作にもAIを使用しています。

③ リグ入れの方針転換

当初は既存アセットのドラゴンのボーンを移植する予定でしたが、 ウェイトなどの問題が発生したため、デザイナーが自力でリグ入れを行う方針に切り替えました。

④ 並列作業で進行維持

リグ作業で工期がずれそうだったため、私はプレイヤーアクションを先にブラッシュアップし、 制作全体の進行が止まらないように対応しました。

⑤ Unity実装と尻尾切断

エクスポートされたFBXに当たり判定を設定し、 さらに尻尾メッシュを分離したモデルをもとに、一定ダメージで尻尾切断する要素を実装しました。

Development Notes

工夫点・こだわり

短工期・少人数での完成度追求

約2週間弱という短い工期を最初から設定し、 ほぼ未経験の2人でどこまで効率化し、完成度を高められるかに挑戦しました。

制作主導と進行管理

私が主導して現実的な制作方法を調べ、 作業の優先順位や役割分担を整理しながら進行しました。

7日でプロトタイプ実装

1週間でプロトタイプ完成まで到達することを目標にし、 実際に当たり判定やHP実装を含むプレイ可能な状態まで持っていきました。

チーム内での役割分担

デザインは2人で話し合って決め、 技構想は私、アニメーション実装はデザイナーが担当する形で進めました。

UIへのこだわり

タイトル、クエスト、リザルトだけでなく、 戦闘中のHPやブリンクゲージも分かりやすく調整し、 視認性と世界観の両立を意識しました。

HP・ブリンクゲージの分かりやすさ

ブリンク残量とHPを戦闘中にすぐ判断できるようにし、 アクションゲームとして必要な情報が直感的に伝わるUIを意識しました。

プレイヤー攻撃の爽快感

プレイヤー攻撃時のVFXやSE、ヒットストップを調整し、 爽快感と視覚的な分かりやすさを両立できるよう意識しました。

理不尽感を減らす戦闘設計

敵の攻撃予兆VFXや当たり判定の調整を行い、 被弾時に納得感が出るように工夫しました。

難易度ごとの差別化

同じモンスターを使っていても使い回し感が減るように、 行動間隔、アニメーション速度、大きさ、天候、雨パーティクルまで調整しました。

尻尾切断ギミック

尻尾を分離したモデルを使い、一定ダメージで尻尾が切断される要素を実装しました。 本作の象徴的なシステムとして重点的に作り込みました。

1フレーム単位の調整

ヒットストップやエフェクトの発生タイミングなど、 アクションの気持ちよさに直結する部分を細かく調整しました。

テストプレイによる改善

10数人にテストプレイしてもらい、スコアや感想を分析して 難易度や演出、遊びやすさを改善しました。

コントローラー対応

バグ修正中ではありますが、キーボードだけでなくパッド操作にも対応し、 アクションゲームとして自然な操作感を目指しています。

After Action Report

制作振り返り・学べたこと

制作振り返り

短期間で完成に持っていくため、最初から工期と中間目標を定めて動いたことは有効でした。 一方で、モデル制作やリグ入れでは想定外の問題も多く、計画通りに進まない場面もありました。

学べたこと

短工期の開発では、完璧な準備よりも「まず遊べる核を早く作ること」が重要だと学びました。 また、問題が起きたときに作業を並列化して進行を止めないことの大切さも実感しました。

次回に活かしたいこと

次回は、制作前に必要素材や想定トラブルも含めて、より具体的な企画・制作計画を立てることで、 さらに安定した進行と高い完成度を目指したいです。
